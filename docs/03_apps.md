# 03 画面仕様(mfg)

実装規約(自己完結型Widget・3状態・go_router・許可パッケージ)は
CLAUDE.md に従う。製品カードは 写真 / 一言(summary)/ 型番(code)/
主要仕様の抜粋 / 価格表記(「要見積」等)の順で統一し、過度な装飾をしない。

## 公開静的サイト(Flutterではない)
- Python(テンプレート)で products から生成。会社案内/分類別カタログ/
  製品ページ/事例。各製品ページに「見積を依頼する」→顧客アプリの
  /p/{code} へ。SEOはここが担う

## 顧客アプリ(Web / iOS / Android)
```
/            カタログホーム(分類グリッド+新着。静的JSONを読む)
/c/:slug     分類別製品一覧(ProductCard: 写真/一言/型番/仕様抜粋/「要見積」)
/p/:code     製品詳細(仕様表・写真・図面DL)+「見積カートに入れる」
             (数量+個別仕様メモ)。QRの飛び先
/cart        見積カート → 「見積を依頼する」(一言要望を添えて送信)
/quotes      依頼一覧(状態バッジ: 依頼中/回答済み/受注/完了)
/quotes/:id  依頼詳細: 品目リスト+状態+メッセージ(添付可)。
             回答受領後に「発注する」ボタン(→ordered。正式契約は商流で)
/q/:no       見積番号ページ(見積書・納品書のQRの飛び先。認証必須、
             GET /quotes/by-no で自分の依頼なら詳細へ。他人の番号は404)
/scan        QR読み取り(mobile_scanner)→ 製品/見積ページへ
/login /register
```
- カタログ系Widgetは静的JSON、依頼系WidgetはAPI。API不応答時は
  error状態で「メンテナンス中」を表示(3状態パターン)

## 社内アプリ(Flet / Python。左ナビ+メインペイン)

backend の models / services を import し DB直結。
各 view が自分でデータ取得と状態を持つ自己完結型(Widget方針のPython版)。
ログインのみ PocketBase(answered_by の本人記録のため)。
実行は会社サーバー上で `flet run --web`(社内LAN / SSHトンネル)。
```
ダッシュボード   未回答件数(赤バッジ)・直近の依頼
見積対応        requested を上に。詳細: 品目・数量・個別仕様・要望
               → 回答(メッセージで金額・納期を提示、answered へ。
               担当者 answered_by を記録)→ 受注(ordered)→ 完了
製品管理        一覧(テーブル)・編集(仕様はキー:値の動的フォーム)・
               公開/非公開・保存で静的カタログ再生成
在庫・価格      基幹API中継の値を製品別に表示(無ければ「要見積」)
エクスポート    見積台帳/受注台帳 xlsx・CSV(OnlyOffice でそのまま台帳に)
QRラベル       選択した製品のQR面付けPDF(銘板・紙カタログ用)
ユーザー管理    顧客の承認・凍結、スタッフ担当名(admin)
```
- スマホからの簡易対応は、Flet の Web 画面をスマホブラウザで開けば足りる
  (専用モバイルアプリは作らない)
