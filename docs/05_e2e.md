## 0. 前提（ターミナル 3 枚）

```bash
cd ~/dev/mfg
scripts/db-init                 # data/mfg.db に schema 適用（何度でも可）
```

| 端末 | コマンド | 役割 |
|---|---|---|
| A | `pb/pocketbase serve` | PocketBase (localhost:8090) |
| B | `scripts/api` | FastAPI (localhost:8000、自動リロード) |
| C | `scripts/staff` | 社内アプリ Flet (localhost:8550) |

顧客アプリは D で: `cd app && flutter run -d chrome`
（API 先は既定で `http://localhost:8000/api/v1`。変える場合は
`--dart-define=API_BASE=...`）

## 1. PocketBase 疎通

- □ ブラウザで http://localhost:8090/_/ を開き、管理者アカウント作成（初回のみ）
- □ `users` コレクションにテストユーザーを 2 人作る
  - 顧客用: `customer@example.com` / パスワード任意
  - スタッフ用: `staff@example.com` / 同上
- □ **判断ポイント**: `users` に `company` フィールド（text）を足すか決める
  - 足す → 顧客レコードに会社名を入れる。`core/auth.py` が `rec.company` を読んで
    `app_users.company_name` に写す
  - 足さない → company_name は NULL のまま動く（後から足しても壊れない）
- □ トークン検証の疎通確認（curl で PB → API の順）:

```bash
# PB から顧客トークンを取る
TOKEN=$(curl -s http://localhost:8090/api/collections/users/auth-with-password \
  -H 'Content-Type: application/json' \
  -d '{"identity":"customer@example.com","password":"<パスワード>"}' | \
  python3 -c 'import json,sys; print(json.load(sys.stdin)["token"])')
echo "PB OK: ${TOKEN:0:20}..."

# API がそのトークンを受けるか（app_users 自動作成が走る）
curl -s http://localhost:8000/api/v1/cart -H "Authorization: Bearer $TOKEN" | head -c 200
```

- □ 期待: 2 本目が 401 にならず JSON が返る。`data/mfg.db` の `app_users` に
  1 行できている:
  `sqlite3 data/mfg.db "SELECT id, display_name, company_name FROM app_users"`

## 3. E2E 一巡（依頼 → staff 回答 → 発注）

顧客アプリ（Chrome）で:

- □ `customer@example.com` でログイン（新規登録経由も一度試す）
- □ カタログから品を選びカートへ → **見積依頼を送信**
- □ （メール設定なしの開発環境では送信スキップのログが API 側に出る想定。
  エラーで落ちないことだけ確認）

社内アプリ（Flet）で:

- □ 受信した依頼が一覧に出る
- □ 単価・納期を入れて**回答**（answered_by に操作者が記録されること）

顧客アプリに戻って:

- □ 回答が見える → **発注**
- □ 発注が staff 側の台帳に反映される
  （xlsx 台帳の生成物は backend/services の出力先を確認）

## 落ちたときの見どころ

| 症状 | 見る場所 |
|---|---|
| API が 500（`app_users has no column named email` 等） | data/mfg.db が古い。`CREATE TABLE IF NOT EXISTS` は既存テーブルを更新しないので `rm data/mfg.db && scripts/db-init` で作り直す（開発 DB のみ。実データでは移行 SQL を書く） |
| API が 401 を返す | PB が起動しているか / `pb_url`（既定 http://localhost:8090） |
| 顧客アプリが API に届かない | Chrome の DevTools Network。CORS なら API 側のログも |
| staff が DB エラー | `scripts/db-init` を実行したか（data/mfg.db） |
| メール系の例外 | 開発環境では未設定が正常。落ちるなら持ち帰り |
